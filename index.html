Общие требования к именованию классов

HTML разметка

  • Для разметки можно использовать теги HTML5.
  • Значения всех атрибутов должны быть заключены в кавычки (").
  • Форматирование HTML иерархическое, отступ — 4 пробела.
  • Если нужно избежать пробелов между тегами, между ними вставляется HTML-комментарий.
  • Не допускается использование тега <p> кроме фрагментов вёрстки редактируемых через WYSIWYG.

Именование классов

Именование классов должно вестись в терминологии БЭМ.

Схема именования: 

.block
.block_mod_value
.block__element
.block__element_mod_value
ru.bem.info/method/definitions/

Использование префиксов

Использовать префиксы разрешается в редких случаях для обозначения

  • общих / служебных / глобальных классов (например, .g-content и .g-clearfix),
  • служебных классов для JS (например, .js-button)

Семантика именования классов

Названия блоков и элементов должны соответствовать их назначению и функции, а не тому, как они отображаются.

Если блоки выполняют разные функции, но выглядят одинаково, их нужно оформить как один блок с нейтральным названием, отражающим их сходство. Нет излишнему дублированию кода.

Плохо
.col-3 {}
Хорошо
.header {}

Применение каскада

Каскад стилей должен быть сведён к минимуму. Он допускается, в частности, в случае, когда нужно оформить редактируемый в wysiwyg контент; этот контент по возможности должен не содержать классов. 

.g-content p {}
.g-content table {}

Требования к стилям

Базовый CSS

Сброс стилей должен осуществляться с помощью normalize.css

normalize.css

Форматирование стилей

Каждое правило должно располагаться в начале новой строки. После правила следует пробел, фигурная скобка, далее на новых строках правила и закрывающая фигурная скобка на новой строке. У свойства после названия двоеточие, пробел, затем значение. Отступы - 4 пробела.

Плохо
.article, h1{
    color:#777;}
Хорошо
.article,
h1 {
    color: #777;
}

Константы

Предпочтительно использование сокращённых форм записи констант:

  • #777 вместо #777777,
  • 0 вместо 0px,
  • padding: 0 10px 5px вместо padding: 0 10px 5px 10px

и т.д.

Порядок CSS-свойств

Приблизительная схема расположения свойств: 

{
    Позиционирование
    Параметры блока
    Размеры
    Таблицы / списки
    Свойства текста
    Шрифт
    Цвет
}
Плохо
.article {
    color: #777;
    font-size: 16px;
    font-style: italic;
    line-height: 30px;
    left: 0;
    margin-top: -30px;
    position: absolute;
    text-align: center;
    top: 50%;
    width: 100%;
}
Хорошо
.article {
    position: absolute;
    top: 50%;
    left: 0;
    width: 100%;
    margin-top: -30px;
    line-height: 30px;
    text-align: center;
    font-size: 16px;
    font-style: italic;
    color: #777;
}

Расположение по файлам

Стили, принадлежащие одному блоку, должны следовать друг за другом: сначала блок, потом его модификаторы (также переопределения для IE, для отключённого JS и т.п.), затем элементы и их модификаторы.

Стили для IE задаются с помощью классов. Правило с хаком должно следовать непосредственно за оригинальным правилом.

CSS должен быть разбит по файлам. В каждом файле должны содержаться стили одного или нескольких родственных блоков. 

.article__header {
    // ...
}
.article__header:hover {
    // ...
}
.ie8 .article__header {
    // ...
}

Использование CSS-препроцессоров

Применение

Допускается (и даже поощряется) использование LESS.

Каждый css-класс должен являться отдельным определением за исключением состояний, псевдоэлементов, media queries и т.п.

Следует использовать переменные, примеси (аккуратно), функции.

lesscss.org
Плохо
.article {
    // ...
    &__header {
        // ...
    }
}
Хорошо
.article {
    // ...

    &:hover {
        // ...
    }
}
.article__header {
    // ...
}

Организация less-кода

  • При использовании less правила организации файлов такие же, как и для обычного css.
  • Общие используемые переменные и миксины должны быть вынесены в отдельные файлы и подключаться по мере необходимости.
  • Переменные для одного блока должны объявляться непосредственно перед ним.
  • Переменные и миксины должны быть названы в формате @header-height: 80px; .inline-block() {}
  • Подключение стилей на сайт должно осуществляться через один less-файл
header.less

footer.less

project.less
    @import "header";
    @import "footer";
